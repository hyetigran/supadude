# A single Final Boss, not 3 Mini-Bosses

ADR-0003 committed to a finite, hand-authored Level with deliberate boss placement, originally scoped (issues #7/#8) as 3 fixed Mini-Boss encounters distributed through the Level, each gated by its own Checkpoint. In practice, a Boss Fight fully pauses auto-run and hands control to a self-contained combat loop (CONTEXT.md Boss Fight) — doing that 3 times across a single ~2-minute playthrough repeatedly interrupts the core dodge-and-run loop the rest of the Level is built around, working against the "long Geometry Dash level" pacing ADR-0003 was aiming for: a Geometry Dash level doesn't stop being itself partway through to become a different game three times. We collapsed the 3 Mini-Bosses into a single Final Boss placed near the end of the Level, immediately before the finish line, with exactly one Checkpoint immediately before it. This is a pure content/placement change — the Boss Fight system itself (telegraph → dodge → Vulnerable Window → Punch) is unchanged, and the Level keeps the same authored Obstacle/Collectible content that used to be split across 3 Acts, now run as one continuous stretch ending in the Final Boss.
